# Beadlo 标准导入文件示例

这个目录是根据你的真实资料文件夹整理出的标准格式示例：

`E:\aa  _商品站\37 手链制作套件\自动博客文件`

以后每个新站点建议都整理成这 7 个文件再上传，这样程序不需要猜测 Excel、Word 或 Markdown 里的字段，导入会更稳定。

## 文件清单

- `site.json`：品牌和网站变量，例如网站名、域名、语言、定位、受众、语气、必须表达、禁止表达。
- `author.json`：作者档案和 `storyBank` 一手经验素材库。
- `knowledge.json`：品类知识库，包括术语、权威事实、FAQ、卖点、异议处理、合规规则。
- `links.json`：内链库，包括专题页、博客、产品页、分类页、信任页。
- `style-reference.json`：HTML 样式参考总说明和参考 HTML 结构。
- `project-instructions.md`：站点级全局写作指令，相当于 Claude Project 指令。
- `keywords.csv`：关键词规划表。

## 使用方式

这些文件是字段格式示例，不一定包含完整数据。正式使用时，把你的完整数据按同样字段补齐，再通过「上传文件管理」上传和写入当前站点。

如果来源文件是 Excel 或 Word，建议先人工整理成这里的标准 JSON/CSV/MD，再导入程序。这样可以避免自动识别时把文件误判为 `ignore`，或把字段写入错误模块。
