# 关键词规划表标准模板

以后不管前期关键词规划表怎么做，最终导入内容生产引擎前，建议统一整理成以下字段。字段名尽量不要改，程序会按这些字段稳定识别。

## 必填字段

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---|---|
| blogid | 文本 | B001 | 文章编号，用于运营排序和追踪 |
| keyword | 文本 | bookshelf diorama kit | 核心关键词，必须填写 |
| urlslug | 文本 | bookshelf-diorama-kit | URL Slug，不要带 `/blog/`，程序会用于文章文件名和发布 slug |
| priority | P1/P2/P3 | P2 | 优先级 |
| articletype | 枚举 | buying-guide | 文章类型，建议直接用程序标准值 |
| targetwordcount | 数字 | 1500 | 目标字数 |
| intent | 文本 | Commercial | 搜索意图 |

## 推荐字段

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---|---|
| secondarykeywords | 文本 | miniature bookshelf kit, book nook diorama | 次要关键词，逗号分隔 |
| variants | 文本 | miniature library diorama kit | 变体词/长尾词，逗号分隔 |
| direction | 长文本 | Explain kit choices, difficulty, display ideas, and buyer decision points. | 文章方向，是提示词质量最关键字段之一 |
| internallinkingurls | 文本 | /pages/diorama-guide/, /collections/book-nook-kits/ | 内链目标，逗号或分号分隔 |
| pillartarget | 文本 | /pages/diorama-guide/ | 首选专题页/支柱页 |
| volume | 数字 | 90 | 搜索量 |
| kd | 数字 | 12 | 关键词难度 |
| cannibalcheck | 文本 | Safe; no direct conflict. | 蚕食检查备注 |

## 文章类型标准值

| 标准值 | 中文含义 | 适用场景 |
|---|---|---|
| buying-guide | 购买指南 | best、guide、kit、recommendation、commercial intent |
| price-guide | 价格指南 | price、cost、worth、budget |
| comparison | 对比评测 | vs、compare、difference、which is better |
| faq | FAQ 问答 | 纯问答型长尾词，通常是 can/does/is/what 开头且只需直接回答 |
| how-to | 操作教程 | how to、step by step、make、build、use |
| pillar | 支柱文章 | complete guide、what is、category hub、全品类总览 |

## CSV 表头推荐

```csv
blogid,keyword,urlslug,priority,articletype,targetwordcount,intent,secondarykeywords,variants,direction,internallinkingurls,pillartarget,volume,kd,cannibalcheck
```

## 一行示例

```csv
B001,bookshelf diorama kit,bookshelf-diorama-kit,P2,buying-guide,1500,Commercial,"miniature bookshelf kit, miniature diorama kits for adults","miniature library diorama kit","Explain diorama scene choices, kit difficulty, display ideas, and how it differs from book nook kits.","/pages/diorama-guide/, /blog/diy-miniature-diorama-kit/","/pages/diorama-guide/",90,12,"Safe; no direct conflict."
```

## 使用建议

- 不建议把 `Schema` 当作 `articletype`。`Article + FAQ` 应该拆开：`articletype=buying-guide`，FAQ 作为文章结构的一部分即可。
- `urlslug` 只写 slug，不写完整 URL。正确：`bookshelf-diorama-kit`；不建议：`/blog/bookshelf-diorama-kit/`。
- `direction` 尽量写具体，不要只写“写一篇指南”。方向越具体，文章越不容易同质化。
- 如果不确定文章类型，优先填 `buying-guide`，纯步骤文填 `how-to`，纯问答文才填 `faq`。
